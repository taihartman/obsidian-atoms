Nightly auto-run changes the product
  • Weak: multi-item invite right after Process. Most nights land 0–1 packing atom. Silent auto-run has no audience.
  • Strong: links accumulate every night. When the cluster is real (≥3 + shared trip/entity), home shows a quiet Together card.
  • Open: title list (phone-friendly backlinks). Bodies stay separate. No merge, no checkboxes, no folder move.
  • Invite hub only at threshold if no durable node exists. Same sparse bar as person hubs.

Why this still helps with nightly filing

Process is not the show.
Auto-run files while you sleep. There is often no human at the moment three packing lines become a set. The product must notice across days, then speak once on calm home.
What compounds every night
Quiet links into Camping / trip entity. No UI. Cost is classify you already pay. On night 3 the cluster becomes visible. Night 1 stays silent (correct).
What the user gets
Not a packing app. A memory shelf: open Yosemite packing and see every related claim title. Tap any row into the real atom.
Rejected
Toast every night · “3 of 7 packed” progress · auto-merge bodies · folder next to other packing lists · permanent Collections tab · checkboxes by default.

Ship order

1. Stronger shared links (no UI)
2. Threshold Together card + open title list
3. Hub invite if no node (person-hub cousin)
4. Optional checkboxes / append much later
