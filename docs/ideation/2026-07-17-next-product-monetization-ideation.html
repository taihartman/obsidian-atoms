Read this first. Atoms already ships a real second-brain loop (dump → file → For you) as a free MIT plugin with BYOK Anthropic. The money path is not “add chat AI.” It is: remove phone key friction, own capture ingress outside Obsidian, and never paywall the vault habit. Prior open ideation (2026-07-16) covered auto-run / reconsider / graph — this run is product/business/platform only.

Grounding context

What the product is today, and what is still missing for revenue.

  • Shape: Obsidian community plugin (atoms 0.6.23, isDesktopOnly: false). File+link + mobile home + resurface stream ship. Capture is constitutionally external (iOS Shortcut / daily bullets).
  • Money today: Zero. Plugin free MIT; user pays Anthropic. SecretStorage keys are device-local (don’t Sync). No accounts, entitlements, or managed proxy.
  • Pain already in product copy: home need_key hero — “Add an API key on this phone so Atoms can file them.” Free polish vs “Uses your Anthropic key” refile already splits cost language.
  • Market: Copilot free BYOK + Plus managed AI; Smart Connections free local + Pro; Drafts owns capture surfaces; companions own ingress, plugins own vault truth. Obsidian culture punishes paywalled cores.
  • Distribution: Pre-community harden largely done; public GitHub + Community scan still open. Privacy story is “titles + captures leave over TLS to Anthropic” — no on-device-AI claim.
  • Not STRATEGY.md — north star lives in docs/architecture.md three legs: Capture | File+link | Resurface.

#1 Atoms Plus — managed keys (phone-first conversion)

axis: monetization conf 85% cx Medium

Keep free forever: full plugin + BYOK Anthropic on any device. Ship Atoms Plus (~$8–12/mo annual-friendly) as a managed Anthropic proxy with hard per-day / per-launch caps (reuse PER_LAUNCH_CAP, backfill estimate→confirm, update-notes batch 15). The conversion moment is already on home: phone users hit need_key because SecretStorage keys don’t Sync. Plus removes “paste sk-ant-… on this phone,” not “unlock For you.”

Basis

direct: home need_key hero + device-local key plane; cost gates and free-vs-paid work copy already ship.
external: Copilot free BYOK + Plus managed; Reflect ~$10/mo all-in AI.
reasoned: conversion is setup friction on the primary dogfood device, not feature envy.

Why it matters

Matches what you already said out loud (“few dollars so they use our key”) and attaches it to a real drop-off already in the UI — without inventing a new product loop.

Downsides

COGS and abuse if caps are soft; privacy optics of “we hold the path to Anthropic”; needs auth + metering you don’t have yet; Community culture will punish if Plus feels like the real product.

#2 Capture companion — widgets & share sheet (plugin stays vault truth)

axis: capture / platform conf 80% cx High

Don’t build a second brain app. Build a thin capture shell: Home Screen widget, Share Sheet, Android intent later — only job is append - thought to today’s daily (or a Sync-friendly inbox note). Monetize the companion (Drafts Pro–style yearly) or bundle with Plus. The free MIT plugin remains file + link + home + For you. Capture stays out of the plugin constitution; the companion productizes the already-external leg.

Basis

direct: architecture Capture leg “still not owned”; shortcut versioning / unpushable iCloud links; full custom mobile app OOS as host.
external: Drafts capture surfaces; Nabu Casa / Zotero Connector pattern — thin ingress, local truth free.

Why it matters

Your “standalone app for widgets + better quick capture” lands cleanly here without forking the vault into a proprietary DB or fighting Community norms.

Downsides

Two apps to install; App Store tax; Sync/URI reliability; Android/iOS dual ship cost; risk of companion scope creep into “mini Obsidian.”

#3 Judgment-budget SKU line — never sell the habit

axis: monetization conf 82% cx Low

Productize the free/paid split already in the quality-refresh brief: local polish, library, For you, protect-existing stay free forever. Money only buys judgment — classify, Update refile, backfill — as a monthly quota or prepaid pack. Home can show a simple “judgment budget” meter that maps to copy users already see (“Free — no API key” vs “Uses your Anthropic key”).

Basis

direct: free polish vs API refile; “free work free, paid work rare / never auto-spend”; backfill cost-estimate modal.
external: Craft AI credits; Day One AI on Gold tier only.

Why it matters

This is the packaging rule for #1. Without it, Plus drifts into “pay to use your notes.” With it, Plus is anxiety insurance for Anthropic spend and key setup — not a second-brain tax.

Downsides

Metering UX can feel like a video-game stamina bar if mis-tuned; prepaid packs need store plumbing; power users may still prefer raw BYOK (good — keep that path first-class).

#4 Paid product = filing uptime (not richer For you)

axis: monetization / core loop conf 70% cx High

Invert “premium AI features.” Free product gets the full home library and For you stream forever. Paid sells unattended filing reliability: past dailies drain when Obsidian isn’t open (proxy worker + caps + same egress ack language), so the phone habit isn’t “open app → Process.” Resurface stays free so the brand never becomes a gated review queue.

Basis

direct: auto-run is device-local and silent; launch caps exist; resurface is free local multi-cue stream.
reasoned: COGS live on classify, not on showing an atom you already own; guilt-free recall is the brand.

Why it matters

Gives Plus a concrete value prop beyond “we hold the key”: your backlog files overnight. Aligns with “phone is the product, desktop is the workshop.”

Downsides

Needs always-on infrastructure beyond the plugin; privacy and reliability bar is high; auto-run productization collides with “never process today” and offline phone realities.

#5 Community listing before any checkout

axis: distribution & trust conf 88% cx Medium

Sequence ruthlessly: finish public repo + Community preview scan + privacy narrative, get strangers installing free BYOK, then ship Plus or companion IAP. Hosted keys before directory listing multiplies “who holds my captures?” distrust. Optional-payments language already lives in the security learning — use it after trust, not as a substitute for distribution.

Basis

direct: pre-community review: harden done, public GitHub + Community scan still open; README MIT + optional paid Anthropic; protect-existing integrity work.
external: Obsidian plugins free-to-install; money lives in companions/cloud after adoption.

Why it matters

Without a funnel, Plus has no buyers. With a free Community install, every later monetization compounds.

Downsides

Delays revenue; freemium users may never convert; public listing invites support load before paid ops exist.

#6 Core-loop depth as conversion fuel (not a detour)

axis: core loop conf 72% cx Medium

Version gravity is already home polish, quality stamps, Update notes, and For you. Keep 1–2 cycles improving file quality and zero-guilt recall so that when Plus or the companion ships, the loop feels worth a bill. Don’t monetize a flaky Process path. Don’t confuse this with “never ship money surfaces” — depth is CAC for later conversion, not a permanent substitute for a SKU.

Basis

direct: 0.5–0.6.x shipping inventory (quality stamp, update strip, 4-cue resurface, land peak); past learnings: stream not Anki, user-gated refile, marker integrity = trust.
reasoned: paid users churn if filing feels wrong; free quality is the demo of Plus.

Why it matters

Prevents the classic indie trap: bill for infrastructure while the daily experience still feels like a science project.

Downsides

Easy to use as an excuse to delay money forever; must pair with a hard sequence (#5 + #1/#2), not open-ended polish.
