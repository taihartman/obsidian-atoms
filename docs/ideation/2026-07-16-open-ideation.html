Grounding context

Repo-grounded surprise-me scan of Obsidian Atoms 0.5.3 — past daily bullets → classify atom/task/noise → flat Atoms/ + markers → mobile home + multi-cue “From the brain” stream.

  • Product gravity (git): home polish + recall (0.5.x), not pipeline greenfield. North star: capture done · file+link core · resurface shipping.
  • Non-negotiables: verbatim body; two write types only; never today by default; SecretStorage; sentinel for all verdicts; stream ≠ Anki queue; no folders / no append-into user notes.
  • Institutional learnings: marker line-drift guards; home-native progress for long API; multi-cue rehearsal not queue; epistemic rot amplified by append-only confident titles; hybrid people/media repair.
  • External white space: almost nobody ships full daily→AI triage→verbatim atoms→markers→soft multi-cue stream. Competitors: Note Refactor (manual), Copilot/Smart Connections (chat), SRS plugins (guilt), Readwise (attention).
  • Fertile subjects discovered: epistemic rot / supersession, same-claim consolidation, cost/context shortlist, orphan/graph hygiene, home as attention surface, graph citizens (people/media).

#1 Belief OS — soft death + citator + mind-change stream

Conf 85% Complexity Medium–High Epistemic rot

Treat changed minds as the product’s load-bearing surface. When reason-bearing links say revises / contradicts, show a subsequent-history ribbon (citator), demote superseded claims from library and For you by default (“soft death” without deleting files), and make the next stream card prefer the old body with a one-line “later you wrote [[new]]” — still Open / Next / soft throttle, never a due queue.

Supersession edge drives rehearsal; soft death demotes fossils without deleting them.

Basis

direct: Spec amendments name epistemic rot as amplified (confident titles, no consolidation, equal-live fossils); cheap anti-rot is supersession-shaped link reasons. Classify already hunts revises/contradicts. Multi-cue stream solution lists supersession-aware as open future.
external: Common-law citators / “subsequent history” — claims evolve without rewriting the record; Readwise-style attention without SRS guilt.

Why it matters

Filing is largely solved. The next daily pain is not “missing notes” — it’s not knowing which version of yourself to trust when For you shows an old claim. This is the product differentiator vs chat-over-vault tools.

Downsides

Soft-death rules can hide history users still want; supersession quality depends on classify link reasons (today uneven). Needs careful empty-stream behavior when no mind-change pairs exist.

#2 Collision → continuous consolidation

Conf 80% Complexity Medium Same-claim clusters

Stop treating same-title collision as plumbing. Process done-summary and home surface “you returned to this claim”; when a cluster grows, optionally create a constellation / MoC atom that links members with reasons — bodies untouched, still only new files. The write path already collapses collision into supersession (no silent Title 1.md); productize that as the funnel into synthesis.

Sediment → spine without rewriting bodies (MoC that links, never merges).

Basis

direct: Amendments H2 / KTD8: same-title collision is the consolidation signal; architecture v2 consolidation “links, never rewrites”; “no consolidation → sediment not synthesis.” Render already updates on collision rather than inventing Title 1.
reasoned: Full batch MoC is scary; collision already fires — harvest it into continuous graph hygiene.

Why it matters

Completes File+link’s missing distillation step inside the two-write model. Turns deferred “v2 consolidation” into progressive enhancement of a path users already hit.

Downsides

Title collision is only one identity signal (misses same claim, different titles). Constellation notes need a clear “plugin-owned” body contract so they don’t fake verbatim captures.

#3 Graph citizens home

Conf 78% Complexity Medium People · media

When the wait queue is clear, home prioritizes hubs — people with new backlinks, cold contacts, media/status objects — with atoms as evidence edges you drill into. Hub attractors receive links only (person pattern generalized); never append into Movies.md or person bios. Collection UIs become read views over flat atoms + tags.

Basis

direct: People hub discovery + post-repair; hybrid media tags + enrichMediaLinks; append cut for lists; design handoff chips not CRM tabs.
external: Letterboxd status objects; CRM “who went cold”; Capacities object types — status without folder filing.

Why it matters

Filing atoms without citizen surfaces is a graveyard with better filenames. Product gravity already moved to graph; clear-state home should show the graph’s people and works, not only a recent-title list.

Downsides

Risk of sliding into mini-CRM (explicitly deferred). Needs sparse-vault empty states. Media hubs must stay link-only to honor append cut.

#4 Attention-first home

Conf 75% Complexity Medium Home UX

Make auto-run (device-local, silent, never-today) the default primary path once key + egress work; demote Process to recovery. Wait card becomes a fault state, not the job list. Prefer body-primary rows; optional Radio mode (stream-first). “Settle today / Process settled” affordance — still never auto-steals mid-capture unmarked live bullets.

Basis

direct: Market pain: evening refactor never happens; home-native progress exists because Process is the main job; design For you / one hero; never-today is a proxy for mid-capture safety, not a forever ban on evening filing.
external: Readwise / Apple Photos For You — attention product, not note browser.

Why it matters

Finishes Capture → File → Resurface psychologically: the user stops re-entering “process my notes.” Every hour on wait-card UX is an hour not spent on graph intelligence and rehearsal.

Downsides

Auto-run default needs rock-solid cost/trust (pairs with idea #5). Settle-today must not silently violate never-touch-live-bullets. Radio mode may anger power users who want the shelf.

#5 Cost design stack

Conf 82% Complexity Medium Scale · ContextProvider

Three coordinated moves: (1) instrument cached-prefix tokens every Process and tripwire shortlist; (2) default ContextProvider neighborhood (mentions, recent seeds, hubs, vocab) instead of all-titles; (3) local pre-gate for pure logistics → noise markers with zero API, reserving Anthropic for claim-/list-/person-shaped text and supersession-quality links. Dry-run shows pre-gate vs model.

Cost tracks belief-shaped work, not raw bullet volume.

Basis

direct: ContextProvider is the one-function shortlist seam; all-titles may cost-scale poorly; architecture “log prefix tokens per run”; hybrid local repair is the enrichment pattern; fully marked vault = 0 API quality bar.
external: Local spam filters vs paid importance models; hosted second brains hide cost — local+API dies on surprise invoices.

Why it matters

The product taxes itself for succeeding: every new atom title re-enters the cold prefix. Without a measured stack, auto-run and phone-primary die as the vault grows.

Downsides

Pre-gate false noise on keepable bullets is a trust cliff — needs dogfood fixtures. Shortlist may miss prior-art titles (mitigate with dry-run “cited priors” transparency).

#6 Soft unfreeze reclassify

Conf 88% Complexity Low–Medium Trust UX

Sentinels correctly freeze task/noise — but a wrong first pass leaves captures in jail, with only tribal knowledge “delete the marker line.” Ship guided Reconsider: dry-run that one capture, show before/after, then replace the plugin-owned marker region only (and create atom if warranted). Still never rewrite capture body; still bottom-up / re-locate / idempotent guards.

Basis

direct: Amendments: manual reclassify = delete marker; non-atom markers freeze verdict; home-native progress / double Process fear; marker-line-drift solution’s multi-edit guards.
external: Priority Inbox undo — pure auto-triage without undo trains distrust.

Why it matters

Idempotency fixed reprocess bombs; the next trust cliff is irreversible-feeling triage. Small surface, high dogfood payoff, unlocks confidence for consolidation and auto-run.

Downsides

Amend-lane energy — less “surprise product vision” than #1–#3. Must not re-open today by default. Marker replace needs careful extent re-locate tests.

#7 Vault health OS

Conf 80% Complexity Medium Recovery

Trial-balance command: count orphan markers (target missing), stacked markers, rebuild-from-daily source (verbatim body) or clear. Extract a Sentinel Job Kit (bottom-up, re-locate, already-has-marker, home progress) that orphan rebuild, consolidation, and backfill healing all implement. Never silent auto-reprocess on unresolved links (KTD10).

Markers are watermarks — unbalanced entries get a reconcile job, not full reprocess.

Basis

direct: Architecture Future #5 orphan rebuild; KTD10 marker authority; marker-line-drift solution; home progress generalizes to long jobs; Sync + multi-device make orphans real.
reasoned / external: Double-entry trial balance — detect loss without rewriting history; ETL watermark reconcile.

Why it matters

Closes the “nothing destroyed” loop when destruction happens outside the plugin (rm, Sync partials). Makes secondary vault jobs inherit the safety envelope instead of re-learning corruption modes.

Downsides

Can feel like an admin console if not framed as trust theater. Rebuild-from-daily may lack title/tags quality without optional re-classify of that capture only.
